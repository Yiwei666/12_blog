# 病毒防护

# 1. 脱机扫描

## 1. Microsoft Defender原理和历史

- 基本原理

> Microsoft Defender 脱机扫描（Offline Scan）通过在操作系统加载前介入，从根本上规避了恶意软件的自我保护机制。
>
> 其核心原理为：触发扫描后，系统强制重启并引导至基于 WinPE 的隔离轻量级执行环境，此时主系统分区处于完全静止状态，恶意进程无法激活、无法调用系统 API 进行伪装或对抗。
>
> 扫描引擎在此可信计算基（TCB）内以最高特权级对磁盘数据进行静态分析与特征匹配，有效应对 Rootkit 等具备运行时隐匿能力的持久化威胁，从而实现常规在线扫描无法达到的深度清除效果。


- 发展历史

> Windows Defender 脱机扫描功能的技术演进历经三个阶段。
> 
> 2011年5月，微软发布原型工具 Microsoft Standalone System Sweeper（MSSS），其底层基于 WinPE 3 构建，以应对 Rootkit 类恶意软件在系统内核层劫持的威胁，采用外部可引导介质实现隔离查杀。
> 
> 同年12月初，该工具以 Beta 版形式更名为 Windows Defender Offline（WDO），并作为独立部署的应急解决方案面向用户发布；然而其操作流程依赖外接存储设备，技术门槛较高。
> 
> 2016年8月2日，随 Windows 10 周年更新（Version 1607）发布，微软将脱机扫描功能作为内置组件集成至操作系统发行版，用户无需创建可启动介质，可直接在 Windows 安全中心一键触发重启查杀，标志着该技术正式完成现代化转型。






## 2. 操作步骤

1. 在 Windows 安全中心 ➡️ “病毒和威胁防护” ➡️ 点击 “扫描选项”。

2. 勾选最下面的 “Microsoft Defender 脱机版扫描” (Offline scan)。

3. 点击“立即扫描”。
> [!NOTE]
> 系统将自动重启，并在 Windows 操作系统完全加载之前的预启动环境（Pre-boot Environment）中执行离线扫描（Offline Scan）。在此阶段，操作系统内核尚未运行，恶意软件无法通过驻留进程或 Rootkit 技术进行自我隐藏或干预扫描进程，从而确保检测结果的准确性与完整性。
> 
> 重启完成后，若系统未生成新的威胁检测报告，则可判定当前系统环境未受恶意软件感染，安全状态正常。

# VLESS + REALITY + XTLS Vision（TCP）方案


# 1. Xray服务器端安装

## 1. 安装和更新

- 安装并升级 Xray-core 和地理数据，默认使用 User=nobody，但不会覆盖已有服务文件中的 User 设置

```sh
bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install

# 设置 Xray 开机自启
systemctl enable xray

# 检查 Xray 运行状态
systemctl status xray
```


- xray配置文件路径

```sh
/usr/local/etc/xray/config.json
```


- 常用命令

```sh
# 查看当前进程
ps -ef | grep -E 'v2ray|xray' | grep -v grep

# 查看端口占用
ss -ltnp | grep -E 'v2ray|xray'

# 查看已有配置路径
systemctl cat v2ray
systemctl cat xray

```

## 2. 配置文件

### 1. xray常用命令行

```sh
# 查看当前安装的 Xray 版本
xray version

# 生成 UUID
xray uuid

# 生成 REALITY 所需的 X25519 密钥对
xray x25519

# 从已有私钥推导公钥
xray x25519 -i "你的私钥内容"

# 校验配置文件语法，不启动服务
xray run -test -c /usr/local/etc/xray/config.json

# 手动以前台方式运行 Xray，用于实时调试
xray run -c /usr/local/etc/xray/config.json

# 输出合并后的配置，适合多配置文件排查
xray run -dump -c /usr/local/etc/xray/config.json

# 生成临时 TLS 证书
xray tls cert

# 测试目标域名的 TLS 握手
xray tls ping example.com

# 计算 TLS 证书链 hash
xray tls certChainHash
```

### 2. 服务器端配置文件

```sh
# 随机生成 16 位 shortId
openssl rand -hex 8

# 查看443端口占用
ss -ltnp | grep ':443'

# 停止 nginx
systemctl stop nginx

# 测试 Xray 配置
xray run -test -config /usr/local/etc/xray/config.json

# 测试网站是否支持 TLS 1.3 + h2
openssl s_client -connect www.apple.com:443 -servername www.apple.com -tls1_3 -alpn h2 </dev/null 2>/dev/null | grep -E 'Protocol|ALPN protocol|Verify return code'

# 看最近错误日志
journalctl -u xray -e

# 实时跟踪日志
journalctl -u xray -f
```



- 以下模板来源：[/Xray-examples/VLESS-TCP-XTLS-Vision-REALITY](https://github.com/XTLS/Xray-examples/tree/main/VLESS-TCP-XTLS-Vision-REALITY)

```json
{
    "log": {
        "loglevel": "debug"
    },
    "inbounds": [
        {
            "port": 443, 
            "protocol": "vless",
            "settings": {
                "clients": [
                    {
                        "id": "", // run `xray uuid` to generate
                        "flow": "xtls-rprx-vision"
                    }
                ],
                "decryption": "none"
            },
            "streamSettings": {
                "network": "tcp",
                "security": "reality",
                "realitySettings": {
                    "dest": "", // A website that support TLS1.3 and h2. You can also use `1.1.1.1:443` as dest
                    "serverNames": [
                        ""    // A server name in the cert of dest site. If you use `1.1.1.1:443` as dest, then you can leave `serverNames` empty, it is a possible ways to bypass Iran's internet speed restrictions.
                    ],
                    "privateKey": "", // run `xray x25519` to generate. Public and private keys need to be corresponding.
                    "shortIds": [// Required, list of shortIds available to clients, can be used to distinguish different clients
                        "", // If this item exists, client shortId can be empty
                        "0123456789abcdef" // 0 to f, length is a multiple of 2, maximum length is 16
                    ]
                }
            },
            "sniffing": {
                "enabled": true,
                "destOverride": [
                    "http",
                    "tls",
                    "quic"
                ],
                "routeOnly": true
            }
        }
    ],
    "outbounds": [
        {
            "protocol": "freedom",
            "tag": "direct"
        }
    ]
}
```

### 3. 服务器分流策略

`VLESS + TCP + REALITY + Vision` 入站 与下面这种 多 `outbound + routing` 分流 是兼容的。Xray 官方配置中支持 freedom、socks 等 outbound 协议；`routing.rules` 会从上到下匹配，命中第一条后转发到对应 `outboundTag`，没有命中时默认走第一个 `outbound`。

```sh
# 客户端测试chatgpt是否经过WARP
https://chatgpt.com/cdn-cgi/trace

# 配置是否通过语法测试
xray run -test -c /usr/local/etc/xray/config.json

# 确认 socks 代理本身可用
curl -4 --socks5-hostname 127.0.0.1:40000 https://api.ipify.org

# 确认直连 IP
curl -4 https://api.ipify.org
```


如下，相比于模板文件，细化了`outbounds`和`routing`参数值，其余部分未变；与v2ray的分流配置关键词语法略有差异，不能直接复制粘贴。


```json
{
  "log": {
    "loglevel": "warning",
    "access": "/var/log/xray/access.log",
    "error": "/var/log/xray/error.log"
  },  
  "inbounds": [
    {
      "port": 443,
      "protocol": "vless",
      "settings": {
        "clients": [
          {
            "id": "318......b3c",            // run `xray uuid` to generate
            "flow": "xtls-rprx-vision"
          }
        ],
        "decryption": "none"
      },
      "streamSettings": {
        "network": "tcp",
        "security": "reality",
        "realitySettings": {
          "dest": "www.cloudflare.com:443",
          "serverNames": [
            "www.cloudflare.com"
          ],
          "privateKey": "wN......mY",        // run `xray x25519` to generate. Public and private keys need to be corresponding.
          "shortIds": [
            "fb......6"                      //  0 to f, length is a multiple of 2, maximum length is 16
          ]
        }
      },
      "sniffing": {
        "enabled": true,
        "destOverride": [
          "http",
          "tls",
          "quic"
        ],
        "routeOnly": true
      }
    }
  ],
  "outbounds": [
    {
      "tag": "default",
      "protocol": "freedom"
    },
    {
      "tag": "socks_out",
      "protocol": "socks",
      "settings": {
        "address": "127.0.0.1",
        "port": 40000
      }
    },
    {
      "tag": "my-tor",
      "protocol": "socks",
      "settings": {
        "address": "127.0.0.1",
        "port": 9050
      }
    }
  ],
  "routing": {
    "rules": [
      {
        "type": "field",
        "outboundTag": "socks_out",
        "domain": [
          "domain:openai.com",
          "domain:chatgpt.com",
          "domain:github.com",
          "domain:bing.com",
          "domain:ip138.com",
          "domain:whoer.net",
          "domain:phind.com",
          "domain:google.com",
          "domain:googleapis.com",
          "domain:gstatic.com",
          "domain:googleusercontent.com",
          "domain:s3-us-west-2.amazonaws.com"
        ]
      },
      {
        "type": "field",
        "outboundTag": "my-tor",
        "domain": [
          "regexp:\\.onion$"
        ],
        "network": "tcp"
      },
      {
        "type": "field",
        "outboundTag": "default",
        "network": "tcp,udp"
      }
    ]
  }
}
```



# 2. windows客户端

xray客户端下载网址：https://github.com/XTLS/Xray-core/releases


## 1. 客户端配置文件


```json
{
    "log": {
        "loglevel": "debug"
    },
    "inbounds": [
        {
            "listen": "127.0.0.1", 
            "port": 10808, 
            "protocol": "socks",
            "settings": {
                "udp": true
            },
            "sniffing": {
                "enabled": true,
                "destOverride": [
                    "http",
                    "tls",
                    "quic"
                ],
                "routeOnly": true
            }
        }
    ],
    "outbounds": [
        {
            "protocol": "vless",
            "settings": {
                "address": "", // IP address
                "port": 443,
                "id": "", // Needs to match server side
                "encryption": "none",
                "flow": "xtls-rprx-vision"
            },
            "streamSettings": {
                "network": "tcp",
                "security": "reality",
                "realitySettings": {
                    "fingerprint": "chrome", 
                    "serverName": "", // A website that support TLS1.3 and h2. If your dest is `1.1.1.1:443`, then leave it empty
                    "publicKey": "", // run `xray x25519` to generate. Public and private keys need to be corresponding.
                    "spiderX": "", // If your dest is `1.1.1.1:443`, then you can fill it with `/dns-query/` or just leave it empty
                    "shortId": "" // Required
                }
            },
            "tag": "proxy"
        }
    ]
}
```

# cmd中运行xray
xray.exe run -c config.json
